# 별땅 - 사용자 플로우 & 핵심 시나리오

문서 버전: v0.1

## 0. 역할 정의

- **Buyer (B)**: 매수 희망자
- **Appraiser (A)**: 감정평가사
- **Broker (Br)**: 공인중개사
- **Admin (Ad)**: 플랫폼 운영자

## 1. 핵심 거래 플로우 (End-to-End)

```
[Br] 매물 등록
   ↓
[A] 매물 발견 → 리뷰 작성·발행 (가격 본인 설정)
   ↓
[B] 매물 검색 → 매물 상세 → 리뷰 N건 잠긴 상태 확인
   ↓
[B] 관심 리뷰 결제 (1~3건 비교)
   ↓
[B] 리뷰 열람 → 의사결정 → "매수 의향서 제출"
   ↓
[Br] 의향서 수신 → [B]와 컨택 → 오프라인 계약
   ↓
[Br] 거래 완료 보고 → 플랫폼 거래 수수료 정산
   ↓
[A][Br][Platform] 자동 분배 정산
```

## 2. 시나리오별 상세

### 시나리오 A — 매수 희망자가 강남 빌라 리뷰를 결제하고 의향서를 낸다

| Step | 화면 | 사용자 액션 | 시스템 동작 |
|---|---|---|---|
| 1 | 메인 (매물 리스트) | "강남구" 검색, 가격대 30~50억 필터 적용 | 매물 카드 N개 노출 (감정 N건 가능 표기) |
| 2 | 매물 상세 | "감정사 리뷰 (3)" 탭 클릭 | 리뷰 카드 3개 노출 (잠금 + 평가사 프로필 + 열람 비용) |
| 3 | 리뷰 카드 | "₩45,000 해제하기" 클릭 | 결제 모달 → PG 호출 |
| 4 | PG 결제 | 카드 정보 입력, 승인 | 결제 성공 → 권한 발급 |
| 5 | 결제 완료 | "매물 보기" 클릭 | 리뷰 잠금 해제, 추정가·근거 노출 |
| 6 | 매물 상세 | "매수 의향서 제출" 클릭 | 의향가/희망 일정 입력 폼 |
| 7 | 의향서 폼 | 작성·제출 | 등록 중개사에게 알림 발송 |
| 8 | 마이페이지 | 의향 내역 확인 | "중개인 회신 대기" 상태 표시 |

### 시나리오 B — 감정평가사가 매물 리뷰를 발행한다

| Step | 화면 | 사용자 액션 | 시스템 동작 |
|---|---|---|---|
| 1 | 감정사 대시보드 | "리뷰 가능 매물" 리스트 확인 | 리뷰 미작성 매물만 필터링 |
| 2 | 매물 상세 (감정사 뷰) | "감정서 작성" 클릭 | 리뷰 작성 폼 진입 |
| 3 | 가치 산정 | 슬라이더로 추정가 입력 (예: 125억) | 실시간 미리보기 |
| 4 | 시장 전망 | 보합 선택 + 분석 의견 작성 | 글자 수 카운트 |
| 5 | 증거 자료 | 현장 사진·등기부 업로드 (최대 10장) | S3 서명 URL 업로드 |
| 6 | 판매 가격 설정 | ₩49,000 입력 → 수수료 안내 (15%) | 정산 예상액 자동 계산 |
| 7 | "감정서 발행하기" | 클릭 | 리뷰 게시 (수정 불가 안내) |
| 8 | 발행 완료 | 매물 페이지로 이동 | 리뷰가 매물 상세에 노출됨 |

### 시나리오 C — 공인중개사가 매물을 등록한다

| Step | 화면 | 사용자 액션 | 시스템 동작 |
|---|---|---|---|
| 1 | 중개사 대시보드 | "+ 매물 등록" 클릭 | 등록 마법사 진입 (4단계) |
| 2 | Step 1: 기본 정보 | 카테고리, 주소, 면적, 가격 | 카카오 주소 API 호출 |
| 3 | Step 2: 사진 업로드 | 메인 사진 + 추가 사진 (최대 20장) | 자동 리사이즈, 썸네일 생성 |
| 4 | Step 3: 상세 설명 | 설명, 침실/욕실, 주차, 준공 연도 | 마크다운 미리보기 |
| 5 | Step 4: 체크리스트 | 현장 답사 완료 / 등기부 확인 / 패키지 연계 | 신뢰도 표시용 메타데이터 |
| 6 | "등록하기" | 클릭 | 매물 게시 + 감정평가사들에게 알림 |

### 시나리오 D — 거래 완료 → 정산

| Step | 액션 | 시스템 동작 |
|---|---|---|
| 1 | Br: 거래 완료 보고 (계약서 사본 업로드) | 거래 상태 `pending_review` |
| 2 | Ad: 거래 검증 (계약서 진위 확인) | 상태 `verified` |
| 3 | B: 거래 수수료 (예: ₩42,500,000) 결제 | 에스크로 보관 |
| 4 | 자동 분배 잡 실행 (T+3영업일) | • 플랫폼 수수료: 5%<br>• 중개인 수수료: 3%<br>• 리뷰어 수수료: 결제 리뷰 작성자에게 추가 보너스 |
| 5 | 각 사용자에게 정산 영수증 발송 | 이메일 + 마이페이지 |

## 3. 화면 전이도 (Sitemap Lite)

```
[랜딩/매물 리스트]
   ├── [매물 상세] ──┬── [리뷰 결제 모달]
   │                 └── [매수 의향서 폼]
   ├── [로그인/회원가입]
   └── [마이페이지]
        ├── (B) [구매한 리뷰] · [의향서 내역] · [거래 내역]
        ├── (A) [발행한 리뷰] · [정산 내역] · [리뷰 작성 폼]
        ├── (Br) [등록 매물] · [수신 의향서] · [거래 정산]
        └── (Ad) [감정사 승인] · [거래 검증] · [신고 처리]
```

## 5. 에러/예외 플로우

| 상황 | 처리 |
|---|---|
| 결제 실패 | 모달 내 에러 노출 + 재시도 버튼, 권한 미발급 |
| 결제 후 리뷰 삭제 (정책상 발행 후 삭제 불가) | 발생 안 함 |
| 감정사 자격 만료 | 신규 리뷰 발행 차단, 기존 리뷰는 "자격 변경" 안내와 함께 노출 유지 |
| 매물 거래 완료 | 매물 상태 `sold`, 신규 리뷰 결제 차단, 기존 리뷰 열람 권한 유지 |
| 의향서 중복 제출 | 동일 매물 중복 제출 방지 (재요청 시 갱신) |
