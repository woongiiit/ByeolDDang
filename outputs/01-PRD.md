# 별땅(ByeolDDang) - 제품 요구사항 정의서 (PRD)

문서 버전: v0.1 (MVP)
작성일: 2026-05-08

---

## 1. 비전 / 미션

- **비전**: 부동산 시장의 정보 비대칭을 전문가 리뷰로 해소한다.
- **미션**: 10억 원 이상 고가 매물의 매수자가 **객관적 데이터로 의사결정**하고, **감정평가사·공인중개사가 정당한 보상**을 받는 프리미엄 마켓플레이스를 제공한다.

## 2. 문제 정의 (Problem)

| # | 문제 | 영향 |
|---|---|---|
| P1 | 고가 매물 가격의 적정성 판단이 어렵다 (호가 ≠ 가치) | 매수자의 투자 리스크 증가 |
| P2 | 정식 감정평가서는 비용·시간 부담이 크다 | 의사결정 지연·기회 손실 |
| P3 | 감정평가사는 정식 평가 외 부수입 채널이 부족하다 | 전문성의 시장 활용 제한 |
| P4 | 공인중개사는 광고비 부담 + 진성 매수자 매칭 어려움 | 거래 비용 증가, 성사율 저하 |

## 3. 솔루션 (Solution)

전문 감정평가사의 **리포트형 리뷰(전문가 소견)** 를 매물별로 수집·판매하고, 거래까지 연결되는 **3-Sided Marketplace**.

핵심 차별점:
- 매물 페이지의 주력 콘텐츠가 "전문가 리뷰" — 단순 매물 카탈로그가 아니다
- "정식 감정평가서"가 아닌 **"전문가 소견(리뷰)"** 으로 법적 경계 명확화
- 거래 시 에스크로 + 수수료 자동 분배

## 4. 타겟 사용자 (Personas)

### 4.1. 매수 희망자 (Buyer) — 투자자 김VIP
- **나이/직업**: 50대 / 자산가, 법인 대표
- **상황**: 강남권 토지·빌딩 매수 검토 중. 호가 신뢰 어렵고 감정평가는 부담
- **니즈**: "이 매물 50억이 적정한가?"에 대한 **복수 전문가의 의견**을 빠르게
- **지불 의향**: 리뷰 1건당 ₩30,000~₩100,000, 최대 3건 비교

### 4.2. 감정평가사 (Appraiser) — 김철수 평가사
- **나이/경력**: 40대 / 감정평가사 자격 12년차
- **상황**: 본업 외 부수입 채널 모색, 자기 브랜딩 욕구
- **니즈**: 탁상감정·시장의견 작성으로 N잡 수익 + 전문가 평판 축적
- **지불 의향**: 플랫폼 수수료 15% 수용 가능 (정식 평가서 대비 진입 장벽 낮음)

### 4.3. 공인중개사 (Broker) — 박성훈 중개사
- **나이/경력**: 40대 / 한남·강남 전문
- **상황**: 고가 매물 광고비 부담, 진성 매수자 식별 어려움
- **니즈**: 검증된(=리뷰를 결제한) 매수자 풀, 광고비 0원, 빠른 클로징
- **지불 의향**: 거래 성사 시 수수료의 일부를 플랫폼/리뷰어에게 분배 OK

## 5. 핵심 가치 제안 (Value Proposition)

| 사용자 | Before (기존) | After (별땅) |
|---|---|---|
| 매수자 | 호가만 보고 결정, 감정평가 800만원·2주 | 리뷰 3건 30만원·즉시 비교 |
| 감정평가사 | 정식 평가만 가능, 영업 부담 | 리뷰 발행만으로 부수입·평판 |
| 공인중개사 | 네이버 광고 + 콜드콜 | 결제 매수자 매칭, 광고비 0 |

## 6. 핵심 기능 요구사항 (Functional Requirements)

### 6.1. MVP 범위 (Must)

| ID | 기능 | 사용자 | 설명 |
|---|---|---|---|
| F-01 | 회원가입/로그인 | 전체 | 이메일 + 역할 선택 (Buyer/Appraiser/Broker) |
| F-02 | 감정평가사 자격 인증 | Appraiser | 자격증 번호·이미지 업로드 → 관리자 승인 |
| F-03 | 매물 등록 | Broker | 사진·주소·평수·가격·설명·체크리스트 |
| F-04 | 매물 검색/리스트 | 전체 | 카테고리·지역·가격대 필터, 정렬 |
| F-05 | 매물 상세 | 전체 | 사진 갤러리·사양·설명·중개인 정보 |
| F-06 | 리뷰 작성 (감정서 발행) | Appraiser | 추정가액·신뢰도·시장전망(강세/중립/약세)·근거·자료첨부·판매가 설정 |
| F-07 | 리뷰 목록 (매물별) | 전체 | 매물 상세에서 해당 매물의 리뷰 N건 표시 (잠긴 상태) |
| F-08 | 리뷰 결제·열람 | Buyer | **토큰(별) 차감**으로 영구 열람 권한 (PG는 충전 시점에만 호출) |
| F-08-1 | 토큰 충전 | Buyer | 4종 패키지 중 선택 → 결제 PG → 잔액 +N별 (1별 = 100원) |
| F-08-2 | 토큰 잔액·내역 | 전체 | 마이페이지·헤더에서 잔액 + 충전·사용·적립 ledger 확인 |
| F-08-3 | 토큰 적립 | Appraiser | 리뷰 판매 시 가격의 85%를 즉시 토큰 잔액으로 적립 |
| F-09 | 매수 의향서 제출 | Buyer | 매물 상세에서 중개인에게 의향 전달 |
| F-10 | 거래 정산 내역 | 전체 | 결제 완료 후 수수료 분배 내역 (총액·플랫폼·중개·리뷰어) |
| F-11 | 마이페이지 | 전체 | 역할별 대시보드 (구매 리뷰/발행 리뷰/등록 매물/정산 내역) |

### 6.2. Phase 2 (Should)

- F-12 리뷰 평판 시스템 (감정사 별점·신고)
- F-13 알림 (매물 신규 리뷰, 결제, 정산)
- F-14 리뷰어 프로필 페이지 (경력·작성 리뷰 수·평균 평점)
- F-15 매물 즐겨찾기·비교
- F-16 1:1 채팅 (중개인 ↔ 매수자)

### 6.3. Phase 3 (Could)

- 구독제 (월 정액 N건 리뷰 열람)
- 상가·빌딩 등 수익형 부동산 카테고리 확장
- 거래 후 후기 리워드
- API 공개 (외부 부동산 포털 연동)

### 6.4. 제외 (Won't, MVP 범위 밖)

- 정식 감정평가서 발급 (법적 책임 회피)
- 직접 부동산 중개 (공인중개사법 회피, 항상 공인중개사 경유)
- 대출 알선·세무 컨설팅

## 7. 비기능 요구사항 (Non-Functional)

| 항목 | 목표 |
|---|---|
| 가용성 | 99.5% (월 약 3.6시간 다운타임 허용) |
| 응답성 | 매물 리스트 p95 < 800ms |
| 보안 | HTTPS 의무, 비밀번호 bcrypt, JWT 인증, 결제 PG 위임 |
| 개인정보 | 주민번호 미수집, 자격증 사본은 서명 URL로만 접근 |
| 접근성 | 모바일 반응형 필수, WCAG AA 지향 |
| 다국어 | MVP는 한국어 단일 |

## 8. 법적/리스크 요구사항

| 영역 | 요구사항 |
|---|---|
| 감정평가법 | 모든 리뷰에 "정식 감정평가서가 아닌 전문가 소견" Disclaimer 고정 노출. 실지조사 여부 명시. |
| 공인중개사법 | 거래 수수료는 "플랫폼 사용료 + 컨설팅 수수료" 명목으로 분리 청구. 직접 중개 금지. |
| 전자상거래법 | 리뷰 결제는 "디지털 콘텐츠 구매" — 열람 즉시 청약철회권 소멸 명시 (열람 전 환불 가능). |
| 자격 인증 | 감정평가사·공인중개사 자격번호 검증 절차 의무 (관리자 수동 승인 → 추후 API 연동) |

## 10. 가정 및 의존성 (Assumptions)

- PG사: **추후 결정 (Hold)** — MVP는 데모 충전 모드 (즉시 성공). 실제 PG는 토스페이먼츠/포트원 후보군 중 선정 예정
- 호스팅: **Railway** (DB / Redis / Backend / Frontend 단일 워크스페이스)
- 감정평가사 자격 검증은 MVP에서 **관리자 수동 승인**, 추후 한국감정평가사협회 API 연동 검토
- 지도/주소: 카카오 주소 API + 카카오 지도 SDK
- 사진 저장: AWS S3 또는 Cloudflare R2 (서명 URL)
- MVP는 **단일 리전(서울)**, **단일 통화(KRW)**
